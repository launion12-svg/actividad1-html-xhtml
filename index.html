1. Documento HTML5 básico

Todo documento HTML5 comienza con <!DOCTYPE html>, que indica al navegador que debe interpretar el documento usando el estándar HTML5. La codificación UTF-8 se declara en el <head> mediante <meta charset="UTF-8">, permitiendo el uso de caracteres especiales de cualquier idioma.

La estructura fundamental de este documento incluye las etiquetas semánticas de HTML5: <header>, <nav>, <main>, <section>, <article>, <aside> y <footer>, que aportan significado estructural al contenido.

2. Diferencias principales entre HTML y XHTML

HTML y XHTML comparten el mismo propósito —definir la estructura de páginas web— pero difieren significativamente en su sintaxis y rigor:

Criterio HTML5 XHTML
Sintaxis Flexible y permisiva Estricta (basada en XML)
Cierre de etiquetas Opcional en algunas etiquetas Obligatorio siempre
Mayúsculas/minúsculas Indiferente Solo minúsculas
Etiquetas vacías <br>, <img> <br />, <img />
Atributos Sin comillas permitido Siempre entre comillas
DOCTYPE <!DOCTYPE html> Declaración DTD compleja
Tipo MIME text/html application/xhtml+xml
Tolerancia a errores Alta (el navegador los corrige) Nula (error = página en blanco)

3. Justificación: cuándo usar cada uno

HTML5 es la elección recomendada para la gran mayoría de proyectos web modernos. Su amplia compatibilidad con todos los navegadores, su flexibilidad y las nuevas etiquetas semánticas lo convierten en el estándar dominante. Es ideal para sitios web, aplicaciones web, CMS como WordPress, y cualquier proyecto donde la productividad y la compatibilidad sean prioritarias.

XHTML tiene sentido cuando el documento debe integrarse en un ecosistema XML (procesado con parsers XML, transformaciones XSLT o intercambio de datos con sistemas externos). Su rigor sintáctico garantiza documentos bien formados, útil en entornos donde la consistencia estructural es crítica.

En la práctica actual, HTML5 ha desplazado a XHTML como estándar principal, y es la opción recomendada por el W3C para el desarrollo web contemporáneo.
